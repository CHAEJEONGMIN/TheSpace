List-API

Curl request

$ curl 'http://localhost:8080/board/list?page=1&size=10&type=t&keyword=1&path=test&categoryId=1' -i -X GET \
    -H '_csrf: dummyCsrfToken' \
    --cookie 'JSESSIONID=example-session-id'

Request cookies

Name Description

JSESSIONID

Authenticated user session ID cookie

Query parameters

Parameter Description

page

Page number to display in the full query results.

size

Number of rows to display search results.

type

Search condition identifier.
Operation 'or' for search criteria.
t: title
c: content
w: writer
u: uuid
r: reply content

keyword

Search Keyword

path

The community to be searched for.
Operation 'and' for search criteria.

categoryId

CategoryId of Search Destinations.
Operation 'and' for search criteria.

Response fields

Field

Type

Description

page

Number

Page number to display in the full query results.

size

Number

Number of rows to display search results.

total

Number

The total number of rows of search results.

start

Number

1 if search results exist, 0 otherwise.

end

Number

Last page of total search results, total / size

prev

Boolean

True if there are more than 2 pages currently being viewed.

next

Boolean

True if page is less than end.

dtoList

Array

Search results row data.

Message API

Message api

This document outlines the WebSocket API for chat messaging.

1. Send Message

Sends a message to a specific chat room.

1.1. WebSocket Endpoint

  • Destination: /app/chat/send

  • Subscription: /topic/chat/room/{roomId}

1.2. Request Body

The client sends a message object to the /app/chat/send destination.

Table 1. Request Payload (MessageDTOs.Text)
Field Type Description

roomId

Long

The ID of the room to send the message to.

content

String

The text content of the message.
Example Request
{
  "roomId": 1,
  "content": "Hello!"
}

1.3. Response Body

Subscribers to /topic/chat/room/{roomId} will receive a ChatMessage object.

Table 2. Response Payload (ChatMessage)
Field Type Description

messageId

Long

Unique identifier for the message within the room.

roomId

Long

The ID of the room where the message was sent.

sender

String

The UUID of the user who sent the message.

content

String

The content of the message.

type

MessageType

The type of the message (e.g., TEXT, IMAGE).

sentAt

LocalDateTime

The timestamp when the message was sent.
Example Response
{
  "messageId": 123,
  "roomId": 1,
  "sender": "testerUUID",
  "content": "Hello!",
  "type": "TEXT",
  "sentAt": "2025-07-08T10:30:00"
}

2. Get Recent Messages

Retrieves the most recent messages from a chat room. This is a user-specific request.

2.1. WebSocket Endpoint

  • Destination: /app/chat/recent

  • Subscription: /user/queue/chat/room/{roomId}

2.2. Request Body

The client sends the roomId as the payload to the /app/chat/recent destination.

Table 3. Request Payload
Field Type Description

roomId

Long

The ID of the room from which to fetch recent messages.
Example Request
1

2.3. Response Body

The server sends a list of ChatMessage objects to the user’s private queue at /user/queue/chat/room/{roomId}.

Response Payload (List<ChatMessage>)

A list containing ChatMessage objects. See the ChatMessage structure in section 1.3.

Example Response
